# PhotoSelector — Канатка

Автоматический отбор лучших фотографий с горнолыжной канатной дороги.

Одна программа. Один компьютер. Запустил — работает.

---

## Что делает

Камера снимает лыжников на кресельном подъёмнике. Программа:

1. Мониторит папку с входящими фото в режиме реального времени.
2. Группирует кадры в серии по времени создания файла.
3. В каждой серии выбирает лучший кадр (3-слойный скоринг).
4. Отбрасывает пустые кресла.
5. Собирает печатные листы (сетка 2×4).
6. Печатает автоматически или сохраняет в тестовом режиме.

---

## Установка

Запустите `PhotoSelector_Setup.exe` и следуйте инструкциям.

После установки:
- Программа в меню Пуск: **PhotoSelector**.
- Рабочие папки создаются автоматически в `workdir/` рядом с программой.
- Веб-интерфейс открывается в встроенном окне (WebView2).

---

## Быстрый старт

1. Запустите **PhotoSelector**.
2. В навбаре нажмите **▶ Запустить** — начнётся мониторинг входящей папки.
3. Кладите фото в `workdir\incoming\` (или настройте камеру на эту папку).
4. Через 3–6 секунд на главной появятся обработанные серии.

Для тестирования с реальными фото:
1. Распакуйте тестовый архив в папку `INBOX\` рядом с программой.
2. Запустите `simulate_camera.bat` — симулятор будет подавать фото с реалистичными задержками.

Подробнее: `docs/user_testing_guide.md`.

## Веб-интерфейс `localhost:8787`

### Навигационная панель

| Элемент | Назначение |
|---|---|
| **Серии** | Главная страница со списком серий |
| **Листы** | Галерея собранных печатных листов |
| **Настройки** | Инженерная страница настроек (требует пароль) |
| **Обновить** | Принудительное обновление страницы |
| **💾 Архив** | ZIP-экспорт с фильтром по дате |
| **Очистка** | Удаление рабочих данных по папкам |
| **Debug** | Включение/выключение score overlay на превью |
| **💾 X.X ГБ** | Индикатор свободного места на диске (всегда виден) |
| **● Мониторинг** | Статус мониторинга + счётчик обработанных серий |
| **▶ Запустить / ■ Стоп** | Запуск и остановка мониторинга входящей папки |

Индикатор диска: зелёный (норма) → жёлтый (мало места) → красный (критически мало, старт заблокирован).

### Страница «Серии»

- **Карточки серий** с превью лучшего кадра, бейджем решения, score-рейтингом.
- **5-звёздочный рейтинг** с подписями: Отлично / Хорошо / Средне / Слабо / Плохо.
- **Бейджи решения:** `auto_selected` (автоматически выбрана) или `⚠ ambiguous` (спорная, требует проверки).
- **Фильтр-вкладки:** Рабочие / ⚠ Спорные / История.
- **Пагинация:** 20 серий на страницу, кнопки страниц + Загрузить ещё.
- **Переключатель размера карточек:** три режима (крупные / средние / мелкие), сохраняется в браузере.
- **Авто-обновление:** когда мониторинг активен, страница обновляется при поступлении новых серий (каждые 3 сек.).
- **Тост-уведомления** вместо alert-диалогов для очистки, ZIP-экспорта, ошибок мониторинга.
- **Новые серии** отображаются вверху (от новых к старым).

### Страница серии (детальный просмотр)

- Все кадры серии с score overlay (если включён Debug).
- Кнопка **Рядом** — просмотр временно́го окна соседних серий.
- **Спасти фото** — скопировать альтернативный кадр в `selected/`.

### Просмотр «Рядом» (Temporal Browser)

- Показывает ±3 соседних серии по времени.
- **Чекбоксы** для пакетного выбора кадров.
- **Batch Rescue** — отправить несколько кадров в `selected/` одним действием.
- Кнопки «Выбрать все» / «Снять».

### Страница «Листы»

- Галерея всех собранных печатных листов (от новых к старым).
- **Клик по превью** — lightbox с полноэкранным просмотром и свайпом.
- **Кнопка печати** на каждом листе (с подтверждением).
- Индикатор режима: **TEST MODE** (жёлтый) или **Рабочий режим** (зелёный).

### ZIP-архив

Кнопка **💾 Архив** в навбаре открывает модал с пресетами:
- **Всё** — все обработанные фото и листы.
- **Сегодня** / **Эта неделя** — фильтр по дате.
- **Свой диапазон** — ввод дат вручную.

ZIP сохраняется на рабочий стол Windows. Включает: отобранные фото + печатные листы.

### Очистка

Модал с выбором папок для удаления:
- Входящие / Лучшие / Худшие / Пустые кресла / Спорные / Листы / Архив / Логи.
- Кнопка «Выбрать все».
- Удаление безвозвратно; карточки серий строятся по отчётам из `logs/`.

## Настройки инженера

Доступны через **Настройки** в навбаре (пароль по умолчанию: `1234`).

| Группа | Параметры |
|---|---|
| **Детекция серий** | Макс. разрыв внутри серии, Ожидание тишины перед разбором |
| **Haar cascade** | Scale factor, Min neighbors, Min size |
| **Веса скоринга** | 7 компонентов: читаемость лица, ракурс, резкость, освещение, кол-во читаемых, качество кадра, улыбка |
| **Пороги** | 15+ порогов: confidence, sharpness, brightness, readability, fallback ceiling |
| **Спорные серии** | Min. разница score (delta), включить manual review |
| **Сеть** | Автосинхронизация, путь к сетевой папке |
| **Печать** | Автопечать, тестовый режим, имя принтера |
| **Листы** | Размер, сетка, отступы, качество JPEG |
| **Мониторинг диска** | Порог предупреждения (ГБ), критический порог (ГБ) |
| **Логирование** | Записывать лог в файл (toggle; изменение после перезапуска) |

Смена пароля — прямо на странице настроек.
Настройки применяются мгновенно (кроме логирования, требует перезапуск).

## Детекция

Двухэтапная система:
- **MediaPipe Face Landmarker** — основной детектор лиц (468 landmarks).
- **Haar upper body cascade** — fallback для людей в шлемах, масках, очках (ловит ~88% серий).

---

## Скоринг

**3-слойная модель:**

```
1. Occupancy gate — есть ли человек в кадре?
2. Quality gate — pass / weak / fail (резкость, экспозиция)
3. Ranking — итоговый score 0-100
```

**7 компонентов ranking:**

| Компонент | Вес |
|---|---|
| Читаемость лица | 30 |
| Ракурс (yaw/pitch) | 15 |
| Резкость лица | 20 |
| Освещение лица | 15 |
| Кол-во читаемых лиц | 10 |
| Качество кадра | 8 |
| Улыбка (бонус) | 2 |

Серии с близким score (< `delta_score`) помечаются как **спорные** и уходят в `ambiguous/` для ручной проверки.

---

## Режим спорных серий

Если разница score между лучшим и вторым кадром меньше порога `delta_score` — серия помечается `ambiguous_manual_review`.
Спорные фото копируются в `workdir/ambiguous/`. В UI отображается вкладка **⚠ Спорные** с количеством таких серий.
Подтверждение через кнопку **Рядом** → batch rescue.

## Стек

- Python 3.11+, Windows, CPU-only
- OpenCV (`opencv-contrib-python`), Pillow, MediaPipe
- watchdog (мониторинг папки)
- pywebview / WebView2 (нативное окно)

## Структура

```
src/
  app.py                Точка входа (pywebview)
  series_browser.py     Веб-интерфейс (~1600 строк)
  watcher.py            Мониторинг входящей папки
  analyzer.py           Анализ фото (детекция + метрики)
  scorer.py             Скоринг (7 компонентов)
  selector.py           Выбор лучшего кадра в серии
  sheet_composer.py     Сборка печатных листов
  export_utils.py       ZIP-экспорт + сетевая синхронизация
  print_utils.py        Печать (Windows mspaint /p)
  badge_utils.py        Score overlay на фото
  config.json           Все настройки
tools/
  camera_simulator.py   Симулятор камеры (для тестов)
tests/                  58 тестов (unittest)
build/                  PyInstaller + Inno Setup
```

## Рабочие папки

```
workdir/
  incoming/     Входящие фото от камеры (сюда кидать фото)
  selected/     Лучшие фото (по одному на серию)
  sheets/       Печатные листы
  discarded/    Пустые кресла
  rejected/     Проигравшие при отборе
  ambiguous/    Спорные серии (ручная проверка)
  archive/      Обработанные фото (после сборки листа)
  logs/         Логи + отчёты серий
```

Все пути настраиваются через веб-интерфейс (Настройки → Рабочие папки).
