Kanatka: логика отсеивания фотографий

Как 7 шагов Lock Zone превращаются в понятный пайплайн из 3 фаз

IN
Папка INBOX → группировка по времени
Фото сортируются по mtime. Если разрыв между соседними ≤ 3 сек — одна серия.
Пауза > 8 сек — новое кресло. Каждая серия = 1 проезд кресла мимо камеры.
Фаза 1 — Детекция: кто-нибудь есть?
1
Детекция лиц (MediaPipe)
Для каждого кадра серии запускаем FaceDetector + FaceLandmarker.
Результат: список лиц с bbox, landmarks, confidence. Если лицо найдено — subject_present = true.
Лицо найдено → шаг 3 Лицо НЕ найдено → шаг 2
2
Fallback: детекция фигуры человека (HOG)
Если лиц нет — пробуем HOG People Detector (OpenCV).
Ищем фигуру в центральной зоне кадра (18-82% по X, 20-95% по Y).
Если confidence ≥ 0.55 — человек есть, но лицо скрыто экипировкой или ракурсом.
Человек найден → subject_present = true Никого нет → серия = empty
X
Пустое кресло
Ни лица, ни фигуры → вся серия в rejected/empty.
Дальше не анализируется.
✓
Кресло занято → оцениваем
Хотя бы на одном кадре серии найден человек.
Серия переходит в фазу оценки.
Фаза 2 — Оценка: насколько хорош каждый кадр?
3
Скоринг кадра (7 критериев, взвешенная сумма)
Каждый кадр получает оценку 0–100 по формуле:

КритерийЧто проверяемВес face_countЕсть ли человек вообще (бинарно)30 face_orientationРакурс лица (yaw/pitch/roll)20 face_sharpnessРезкость зоны лица15 face_visibilityВидимость лица (перекрытия)15 eyes_openОткрыты ли глаза (EAR)10 face_exposureОсвещение лица5 overall_sharpnessОбщая резкость кадра5
4
Специальные правила экипировки
Горнолыжные очки / маска / шлем — это нормально, не штрафуем.
Если goggle_score или mask_score ≥ 0.45 → eyes_open = 1.0 (не снижаем за закрытые глаза).

Посторонний предмет перед лицом — это плохо, штрафуем.
occlusion_score минус то, что объясняется экипировкой = чистый штраф видимости.
5
Fallback внутри серии
Если в серии есть кадры с лицами и кадры без лиц:
Кадр без лица заимствует лицевые оценки у ближайшего кадра с лицом (с множителем ×0.9).
Это позволяет не терять кадр, где человек есть, но MediaPipe промахнулся.

Если в серии лиц нет совсем, но HOG нашел человека:
Используем selection_score = occupancy × 0.7 + sharpness × 0.3 — упрощенный скоринг без лицевых метрик.
Фаза 3 — Выбор: какой кадр лучший?
6
Лучший кадр серии → selected
Кадр с максимальным score среди всех живых кадров → копируется в workdir/selected/.
Остальные кадры → workdir/rejected/ с бейджем score.
7
Печатный лист (при накоплении 8 фото)
Когда в selected/ набирается 8 фото → собирается лист-сетка 2×4.
Лист → workdir/sheets/, фото → workdir/archive/.

Текущий Lock Zone (7 шагов)

  • 1. Есть ли человек? Нет → empty
  • 2. Есть ли хотя бы один человек? Да → дальше
  • 3. Есть ли читаемое лицо? → лицевые метрики
  • 4. Лицо закрыто очками/маской → не штрафовать
  • 5. Лицо перекрыто предметом → штрафовать
  • 6. Лицо не найдено, но человек есть → осторожно
  • 7. Выбор лучшего кадра в серии

Проблемы: шаги смешивают три разных процесса (детекцию, скоринг, выбор), дублируют друг друга, и их нельзя читать как последовательность — шаги 4-5 происходят внутри шага 3, а шаг 6 — это ветка для случая, когда шаг 3 не сработал.

Предложение: 3 фазы, 7 шагов

  • 1. Ищем лица (MediaPipe) ДЕТЕКЦИЯ
  • 2. Нет лиц → ищем фигуру (HOG fallback) ДЕТЕКЦИЯ
  • 3. Считаем score по 7 критериям ОЦЕНКА
  • 4. Экипировка ≠ штраф, посторонний предмет = штраф ОЦЕНКА
  • 5. Fallback внутри серии (заимствование / упрощенный скоринг) ОЦЕНКА
  • 6. Лучший score → selected, остальные → rejected ВЫБОР
  • 7. 8 selected → печатный лист ВЫБОР

Улучшения: чётко разделены 3 фазы (детекция → оценка → выбор), каждый шаг делает ровно одну вещь, нет дублирования, fallback стал явным шагом.
