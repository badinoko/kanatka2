# Docs Index

Корень `docs/` не является каноническим дашбордом проекта. Для состояния продукта и handoff всегда начинать с:

- `docs/project/overview.md`
- `docs/project/roadmap.md`
- `docs/project/startup.md`
- `docs/project/progress.md`

Что лежит в root `docs/`:

- `user_testing_guide.md` — краткое руководство для пользователя/инженера по установке и полевому тестированию.
- `test_photo_generation_guide.md` — внутренняя шпаргалка по генерации синтетических тестовых серий.
- `pipeline_diagram.html` — визуальная схема текущего pipeline.
- `superpowers/` — внутренние design/spec материалы, не канонический source of truth по текущему состоянию.

Где искать старые raw-заметки и обсуждения:

- `archive/project/2026-03-15/`

Если документ в `docs/` противоречит `docs/project/*`, верить нужно `docs/project/*` и текущему коду.
